# US-007 — Створення профілю дитини

## User Story
**Як** батько/мати,
**я хочу** створити профіль дитини з ім'ям та віковою групою,
**щоб** контент адаптувався під рівень розвитку моєї дитини.

## Acceptance Criteria

### AC-1: Крок 1 — Ввід імені дитини
- **Given** пройдено вибір мови (US-006)
- **When** бачу екран `/(main)/onboarding/name`
- **Then** layout:

```
┌────────────────────────────────┐
│  ← (back → language)           │  ← стрілка назад
│                                │
│     [Маскот з блокнотиком]     │  ← 100dp, ілюстрація
│                                │
│   Як звати малюка? 📝          │  ← AppText h1, center
│                                │
│  ┌──────────────────────────┐  │
│  │                          │  │  ← TextInput, великий
│  │     Максимко             │  │     висота: 56dp
│  │                          │  │     font-size: 24sp (великий!)
│  └──────────────────────────┘  │     border-radius: xl (24)
│                                │     text-align: center
│   0/20 символів                │  ← caption, textSecondary, right
│                                │
│  ┌──────────────────────────┐  │
│  │        Далі  →           │  │  ← AppButton primary, fullWidth
│  └──────────────────────────┘  │     disabled поки ім'я < 2 символи
│                                │
│   Крок 1 з 3    ● ○ ○         │  ← step indicator, center
└────────────────────────────────┘
```

- **And** placeholder: "Введіть ім'я" (зникає при фокусі)
- **And** клавіатура відкривається автоматично (autoFocus: true)
- **And** максимум 20 символів (counter оновлюється)
- **And** кнопка "Далі" стає активною при ≥ 2 символах
- **And** дозволені символи: літери (укр, англ, будь-які), пробіли, апостроф, дефіс
- **And** НЕ дозволені: цифри, спеціальні символи (@, #, $, тощо)
- **And** при натисканні "Далі" → trim пробілів, перша літера uppercase
- **And** KeyboardAvoidingView

### AC-2: Step indicator
- **Given** онбординг flow
- **When** на будь-якому кроці
- **Then** внизу екрану показано progress dots:
  - Крок 1 (ім'я): `● ○ ○`
  - Крок 2 (вікова група): `● ● ○`
  - Крок 3 (аватар): `● ● ●`
- **And** dots: filled = `primary`, empty = `textDisabled`
- **And** розмір dot: 8dp, spacing: 8dp
- **And** padding bottom: 24dp (above safe area)

### AC-4: Деталі картки персонажа

Кожна картка має layout:
```
┌──────────────────────────────────────────┐
│  [Ілюстрація]  │  Коко                   │  ← ім'я: AppText h2, bold
│   персонажа    │  до 4 років              │  ← вік: AppText body, textSecondary
│   60x60dp      │  "Привіт! Я Коко!"      │  ← фраза: AppText caption, italic
└──────────────────────────────────────────┘
```

- **And** ілюстрація зліва: placeholder (emoji великий 40sp або PNG)
- **And** текст справа: вертикально по центру
- **And** фон картки: `characterColor` з opacity 0.08

Кольори персонажів:
| Персонаж | Колір фону (opacity 0.08) | Акцент |
|----------|--------------------------|--------|
| 🐣 Коко | `#FFD93D` (жовтий) | `#F4A300` |
| 🐼 Бамбі | `#90EE90` (зелений) | `#2E8B57` |
| 🦊 Ліса | `#FFB347` (оранжевий) | `#E8721A` |
| 🦉 Софі | `#C4A6FF` (фіолетовий) | `#7B5EA7` |

### AC-5: Анімація вибору персонажа
- **Given** сітка з 4 картками
- **When** тапаю на картку
- **Then** анімація:
  1. Обрана картка: border → 2px `accent color`, scale 1.03 (withSpring, 300ms)
  2. Інші картки: opacity 0.4, scale 0.98 (withTiming, 200ms)
  3. Звуковий ефект: короткий "pop" звук (200ms)
  4. Озвучка персонажа: "Привіт! Я Бамбі!" (аудіо файл, Edge TTS)
     - Відтворюється тільки якщо звук увімкнений в settings
     - Для 🐣 малюків: ОБОВ'ЯЗКОВО відтворюється (voiceRequired: true)
  5. Через 600ms після тапу → автоматичний перехід до аватара (US-008)

### AC-6: Профіль збережено
- **Given** ім'я та вікова група обрані
- **When** перехід до наступного кроку
- **Then** профіль створюється в childProfilesStore:

```typescript
{
  id: crypto.randomUUID(),   // або uuid package
  name: "Максимко",
  ageGroup: "doskilnyata",   // 'malyuki' | 'doskilnyata' | 'pershoklas' | 'drugoklas'
  avatar: null,              // буде обрано на наступному кроці (US-008)
  createdAt: "2026-04-13T10:30:00Z",
  progress: {}               // пустий — заповнюється при грі
}
```

- **And** збережено в MMKV через persist middleware
- **And** `activeProfileId` встановлено на новий профіль

### AC-7: Кілька профілів
- **Given** вже є 1-3 профілі дитини
- **When** батько додає нового через `/(parent)/profiles`
- **Then** запускається той самий flow (name → age-group → avatar)
- **And** максимум 4 профілі
- **And** при спробі створити 5-й → повідомлення: "Максимум 4 профілі. Видаліть один, щоб додати новий."
- **And** кожен профіль має окремий прогрес

### AC-8: Валідація імені
- **Given** поле вводу імені
- **When** вводжу некоректне значення
- **Then** валідація:

| Ситуація | Поведінка |
|---------|-----------|
| Менше 2 символів | Кнопка "Далі" disabled |
| Більше 20 символів | Input не приймає більше |
| Тільки пробіли | Кнопка "Далі" disabled (trim = пустий рядок) |
| Цифри/спецсимволи | Ігноруються (не додаються до input) |
| Emoji | Ігноруються |

## Технічні нотатки
- UUID: `crypto.randomUUID()` (Web + React Native 0.72+) або `uuid` package як fallback
- Вікова група визначає: розміри UI (ageGroups config), складність ігор, озвучку
- Персонажі на MVP — placeholder (великий emoji або прості SVG, замінити на ілюстрації пізніше)
- Озвучка: Edge TTS pre-generated MP3 (ukr voice "Ostap" або "Polina")
- При додаванні нового профілю через parent panel — той самий onboarding flow, але без вибору мови
