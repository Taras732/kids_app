# US-004 — Реєстрація батьків

## User Story
**Як** батько/мати,
**я хочу** створити акаунт за допомогою email та пароля,
**щоб** мій прогрес дитини зберігався і я міг(ла) контролювати використання додатку.

## Acceptance Criteria

### AC-1: Доступ до екрану реєстрації
- **Given** додаток відкрито, користувач не авторизований
- **When** бачу екран логіну
- **Then** є посилання "Ще немає акаунту? Створити" внизу екрану
- **When** натискаю на посилання
- **Then** перехід на `/(auth)/register` з анімацією slide-left (300ms)

### AC-2: Layout екрану реєстрації
- **Given** екран `/(auth)/register`
- **When** бачу екран
- **Then** layout зверху вниз:

```
┌────────────────────────────────┐
│  ← (back button)               │  ← Повернутись на логін
│                                │
│     [Ілюстрація маскота]       │  ← Коко з табличкою "Вітаю!"
│      висота: 120dp              │     або generic welcome image
│                                │
│   Створіть акаунт              │  ← AppText h1, center
│   для батьків 👋               │  ← AppText body, center, textSecondary
│                                │

│  │ 📧  Email                │  │  ← TextInput з іконкою зліва
│  └──────────────────────────┘  │
│  (помилка під полем якщо є)    │  ← AppText caption, color: error

│  ┌──────────────────────────┐  │
│  │ 🔒  Пароль               │  │  ← TextInput + eye toggle справа
```

- **And** фон: `theme.background` (#FFF8F0)
- **And** scroll: якщо клавіатура відкрита — форма скролиться вгору (KeyboardAvoidingView)
- **And** ілюстрація маскота: зменшується/ховається коли клавіатура відкрита
- **And** safe area: padding top для notch/dynamic island

### AC-3: Поля форми — деталі
**Email:**
- Placeholder: "email@example.com"
- Тип клавіатури: `email-address`
- AutoCapitalize: `none`
- AutoComplete: `email`
- AutoCorrect: `false`
- Trim пробілів при submit
- Іконка зліва: 📧 (16dp, textSecondary)

**Пароль:**
- Placeholder: "Мінімум 8 символів"
- SecureTextEntry: true (за замовчуванням)
- Toggle eye іконка справа: 👁 / 👁‍🗨 — показати/сховати пароль
- AutoComplete: `new-password`
- Підказка під полем: "Мінімум 8 символів" (caption, textSecondary)

**Повторіть пароль:**
- Placeholder: "Повторіть пароль"
- SecureTextEntry: true
- AutoComplete: `new-password`
- Підказка під полем з'являється тільки при помилці

**Стилі полів:**
- Висота: 52dp
- Border: 1.5px solid `#DFE6E9` (default) / `primary` (focus) / `error` (помилка)
- Border-radius: `md` (12)
- Padding horizontal: 16dp
- Background: `surface` (#FFFFFF)
- Font size: 16sp
- Іконка зліва: 24dp, padding right 8dp

### AC-4: Валідація
- **Given** форма реєстрації
- **When** взаємодію з полями
- **Then** валідація працює:

| Правило | Коли перевіряється | Повідомлення |
|---------|-------------------|-------------|
| Email пустий | onBlur + onSubmit | "Введіть email" |
| Email невалідний формат | onBlur + onSubmit | "Введіть коректний email" |
| Пароль пустий | onBlur + onSubmit | "Введіть пароль" |
| Пароль < 8 символів | onBlur + onSubmit | "Мінімум 8 символів" |
| Паролі не збігаються | onBlur на confirm + onSubmit | "Паролі не збігаються" |

- **And** помилки показуються під відповідним полем: `AppText caption, color: error`
- **And** поле з помилкою має border color: `error`
- **And** кнопка "Створити акаунт" disabled поки є помилки валідації
- **And** валідація email: regex `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
- **And** валідація в реальному часі: після першого blur на кожному полі, далі onChange

### AC-5: Submit — процес реєстрації
- **Given** форма валідна
- **When** натискаю "Створити акаунт"
- **Then** flow:
  1. Кнопка переходить в стан `loading` (spinner, не натискається)
  2. Виклик `supabase.auth.signUp({ email, password })`
  3. **Якщо успіх** → перехід на екран підтвердження email (AC-6)
  4. **Якщо помилка** → показати помилку (AC-7)
  5. Кнопка повертається в нормальний стан

### AC-6: Екран підтвердження email
- **Given** реєстрація успішна
- **When** бачу екран підтвердження
- **Then** layout:

```
┌────────────────────────────────┐
│                                │
│     [📩 ілюстрація листа]     │  ← 80dp, по центру
│                                │
│   Перевірте вашу пошту!        │  ← AppText h1, center
│                                │
│   Ми надіслали лист на         │  ← AppText body, center
│   user@email.com               │  ← bold, primary color
│                                │
│   Натисніть посилання в листі  │  ← AppText body, center, textSecondary
│   щоб підтвердити акаунт       │
│                                │
│  ┌──────────────────────────┐  │
│  │    Відкрити пошту        │  │  ← AppButton primary (Linking.openURL mailto:)
│  └──────────────────────────┘  │
│                                │
│  Не отримали лист?             │  ← AppText caption
│  Надіслати ще раз (60с)        │  ← link, disabled 60 sec cooldown
│                                │
│  Повернутись до логіну         │  ← link → /(auth)/login
└────────────────────────────────┘
```

- **And** "Надіслати ще раз" — cooldown 60 секунд (показує таймер "Надіслати ще раз (42с)")
- **And** після resend: `supabase.auth.resend({ type: 'signup', email })`
- **And** "Відкрити пошту": `Linking.openURL('mailto:')` — відкриває поштовий клієнт

### AC-7: Помилки сервера
- **Given** виклик signUp завершився помилкою
- **When** отримую помилку від Supabase
- **Then** маппінг помилок:

| Supabase error | Повідомлення користувачу | Де показується |
|---------------|------------------------|---------------|
| `User already registered` | "Цей email вже зареєстровано. Увійдіть або відновіть пароль." | Alert зверху екрану |
| `Signup disabled` | "Реєстрація тимчасово недоступна" | Alert зверху |
| Network error / timeout | "Немає з'єднання з інтернетом. Перевірте підключення." | Alert зверху |
| Rate limit exceeded | "Забагато спроб. Спробуйте через кілька хвилин." | Alert зверху |
| Unknown error | "Щось пішло не так. Спробуйте пізніше." | Alert зверху |

- **And** alert: повнаширинний, зверху форми, фон `error` з opacity 0.1, border-left 4px `error`
- **And** alert автоматично зникає через 5 секунд (або при тапі на ✕)
- **And** для "вже зареєстровано" — посилання "Увійти" веде на `/(auth)/login`

### AC-8: UI стиль — дитячий додаток
- **Given** екран реєстрації
- **When** оцінюю дизайн
- **Then** стиль відповідає дитячому додатку:
  - Кольори яскраві, теплі (theme palette)
  - Маскот/ілюстрація у верхній частині
  - Rounded corners скрізь (border-radius ≥ 12)
  - Великі тапабельні елементи (≥ 48dp)
  - Без корпоративного вигляду
  - Мова проста, без технічного жаргону ("акаунт", "пароль" — не "credentials", "authenticate")

## Технічні нотатки
- Форма: простий `useState` для кожного поля + валідація (без react-hook-form для простоти)
- KeyboardAvoidingView: `behavior="padding"` на iOS, `behavior="height"` на Android
- Supabase signUp повертає `{ data, error }` — завжди перевіряти error
- Email trimming: `email.trim().toLowerCase()` перед відправкою
- Після успішної реєстрації НЕ робити автоматичний логін — чекати підтвердження email
