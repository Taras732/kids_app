# US-005 — Логін та відновлення паролю

## User Story
**Як** зареєстрований батько/мати,
**я хочу** увійти в додаток або відновити пароль,
**щоб** отримати доступ до профілів дітей та їхнього прогресу.

## Acceptance Criteria

### AC-1: Layout екрану логіну
- **Given** додаток відкрито, користувач не авторизований
- **When** бачу екран `/(auth)/login`
- **Then** layout зверху вниз:

```
┌────────────────────────────────┐
│                                │
│     [Логотип Школярик]         │  ← логотип або маскот, 100dp
│                                │
│   Раді вас бачити! 👋          │  ← AppText h1, center
│                                │
│  ┌──────────────────────────┐  │
│  │ 📧  Email                │  │  ← TextInput з іконкою
│  └──────────────────────────┘  │
│                                │
│  ┌──────────────────────────┐  │
│  │ 🔒  Пароль               │  │  ← TextInput + eye toggle
│  └──────────────────────────┘  │
│                                │
│         Забули пароль?         │  ← link, align right, textSecondary
│                                │
│  ┌──────────────────────────┐  │
│  │        Увійти            │  │  ← AppButton primary, fullWidth
│  └──────────────────────────┘  │
│                                │
│  Ще немає акаунту? Створити   │  ← link → /(auth)/register
│                                │
└────────────────────────────────┘
```

- **And** фон: `theme.background`
- **And** KeyboardAvoidingView: форма скролиться при відкритій клавіатурі
- **And** при повороті (landscape) — layout адаптується (ілюстрація зменшується)
- **And** стилі полів: ідентичні з екраном реєстрації (US-004 AC-3)

### AC-2: Поля форми логіну
**Email:**
- Placeholder: "email@example.com"
- AutoCapitalize: `none`, AutoComplete: `email`, AutoCorrect: `false`
- Тип клавіатури: `email-address`

**Пароль:**
- Placeholder: "Ваш пароль"
- SecureTextEntry: true
- Eye toggle справа (показати/сховати)
- AutoComplete: `current-password`

### AC-3: Валідація логіну
- **Given** форма логіну
- **When** натискаю "Увійти" з невалідними даними
- **Then** валідація:

| Правило | Повідомлення |
|---------|-------------|
| Email пустий | "Введіть email" |
| Email невалідний | "Введіть коректний email" |
| Пароль пустий | "Введіть пароль" |

- **And** кнопка "Увійти" disabled поки email та пароль пусті
- **And** помилки показуються під відповідним полем

### AC-4: Успішний логін
- **Given** валідні email та пароль
- **When** натискаю "Увійти"
- **Then** flow:
  1. Кнопка → стан `loading` (spinner)
  2. `supabase.auth.signInWithPassword({ email, password })`
  3. Session зберігається в MMKV (автоматично через adapter)
  4. Перевірка: є профілі дітей в local store?
     - **Якщо є** → navigate до `/(main)/` (Hub)
     - **Якщо немає** → navigate до `/(main)/onboarding/language`
  5. Анімація переходу: fade (300ms)

### AC-5: Помилки логіну
- **Given** невалідні credentials або проблеми мережі
- **When** натискаю "Увійти"
- **Then** маппінг помилок:

| Supabase error | Повідомлення | Дія |
|---------------|-------------|-----|
| `Invalid login credentials` | "Невірний email або пароль" | Alert зверху форми |
| `Email not confirmed` | "Підтвердіть email перед входом" | Alert + link "Надіслати лист ще раз" |
| Network error | "Немає з'єднання з інтернетом" | Alert |
| Rate limit | "Забагато спроб. Спробуйте через кілька хвилин." | Alert |
| Unknown | "Щось пішло не так. Спробуйте пізніше." | Alert |

- **And** alert стиль: ідентичний з US-004 AC-7
- **And** після "Email not confirmed" → кнопка "Надіслати лист ще раз" → `supabase.auth.resend()`

### AC-6: Persist session (auto-login)
- **Given** успішний логін
- **When** закриваю і заново відкриваю додаток
- **Then** session зберігається в MMKV
- **And** автоматично відновлюється при запуску (supabase adapter)
- **And** користувач одразу потрапляє на Hub (без екрану логіну)
- **And** якщо session expired → refresh token автоматично (autoRefreshToken)
- **And** якщо refresh теж expired → redirect на логін з повідомленням "Сесія закінчилась, увійдіть знову"

### AC-7: Layout екрану "Забули пароль?"

- **Then** layout:

│  │ 📧  Email                │  │

│                                │
│  ┌──────────────────────────┐  │

### AC-8: Відновлення пароля — flow
- **Given** валідний email введено
- **When** натискаю "Надіслати інструкції"
- **Then** flow:
  1. Кнопка → loading
  2. `supabase.auth.resetPasswordForEmail(email, { redirectTo: 'shkolyaryk://auth/reset' })`
  3. **Завжди** показується повідомлення: "Якщо акаунт з таким email існує, ми надіслали інструкції"
     (не розкриваємо чи email зареєстровано — безпека)
  4. Layout змінюється на confirmation:

```
┌────────────────────────────────┐
│                                │
│     [📩 ілюстрація]           │
│                                │
│   Перевірте пошту!             │  ← h1
│                                │
│   Якщо акаунт з email          │  ← body, textSecondary
│   user@email.com існує, ви     │
│   отримаєте інструкції для     │
│   відновлення пароля.          │
│                                │
│  ┌──────────────────────────┐  │
│  │    Повернутись до логіну │  │  ← AppButton primary
│  └──────────────────────────┘  │
│                                │
│  Надіслати ще раз (60с)        │  ← cooldown timer
└────────────────────────────────┘
```

- **And** cooldown на resend: 60 секунд

### AC-9: Logout
- **Given** авторизований користувач
- **When** натискаю "Вийти" в батьківській панелі (/(parent)/settings)
- **Then** flow:
  1. Confirmation dialog: "Ви впевнені, що хочете вийти?"
     - Кнопки: "Скасувати" (secondary) | "Вийти" (primary, error color)
  2. `supabase.auth.signOut()`
  3. MMKV session очищується (автоматично через adapter)
  4. Navigate до `/(auth)/login`
  5. Анімація: fade (300ms)
- **And** local data (профілі, прогрес) НЕ видаляється при logout
  (зберігається для можливого повторного входу)

### AC-10: Auth state listener
- **Given** `app/_layout.tsx` (root layout)
- **When** додаток запускається
- **Then** працює auth listener:

```typescript
supabase.auth.onAuthStateChange((event, session) => {
  if (event === 'SIGNED_IN') → check profiles → Hub or Onboarding
  if (event === 'SIGNED_OUT') → Login screen
  if (event === 'TOKEN_REFRESHED') → оновити session в store
  if (event === 'USER_UPDATED') → оновити user в store
});
```

- **And** splash screen показується поки перевіряється initial session
- **And** немає "мерехтіння" між login та hub при запуску (splash ховається тільки після resolve)

## Технічні нотатки
- `supabase.auth.onAuthStateChange()` — підписка в useEffect root layout, cleanup при unmount
- Deep link `shkolyaryk://auth/reset` для password reset callback (потрібна конфігурація в app.json)
- Splash screen: `expo-splash-screen` — `SplashScreen.preventAutoHideAsync()` на початку, `.hideAsync()` після auth resolve
- Не показувати login screen якщо session валідна (prevent flash)
- Confirmation dialog: `Alert.alert()` на mobile, custom modal на web
