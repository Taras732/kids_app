# US-010 — Головний екран з островами

## User Story
**Як** дитина,
**я хочу** бачити головний екран з 8 островами і своїм прогресом,
**щоб** обирати що хочу вивчати сьогодні.

## Acceptance Criteria

### AC-2: Header — привітання та прогрес
- **Given** Hub завантажено
- **When** бачу header
- **Then** елементи:

| Елемент | Деталі |
|---------|--------|
| Аватар | `AvatarCircle` 32dp, зліва |
| Привітання | "Привіт, {name}!" — AppText h2 |
| Рівень | "Рівень {level}" — AppText caption, textSecondary |
| XP stars | "⭐×{totalXP}" — поруч з рівнем |
| Progress bar | XP до наступного рівня: `ProgressBar` color=secondary, height=6 |
| XP label | "{currentXP}/{requiredXP} XP" — caption, під баром |

- **And** формула рівнів: Level 1 = 0 XP, Level 2 = 100 XP, Level 3 = 250 XP, Level 4 = 500 XP, Level N = `50 * N * (N+1) / 2`
- **And** при level up → анімація: число рівня збільшується (scale 1→1.5→1, gold color, 500ms) + звук

### AC-3: Персонаж-компаньйон
- **Given** Hub завантажено
- **When** бачу персонажа
- **Then** відображення:

| Параметр | Значення |
|----------|---------|
| Розмір | 60dp |
| Позиція | Зліва від сітки островів, або над нею |
| Idle анімація | Моргає кожні 3-5с (opacity eyes 1→0→1, 200ms), легкий "дихання" (scale 0.98↔1.02, 2s loop) |
| Speech bubble | "Що вивчимо сьогодні?" — з'являється при завантаженні, зникає через 3с (fadeOut) |

- **And** при тапі на персонажа → випадкова фраза:
  - "Давай пограємо!"
  - "Що вивчимо сьогодні?"
  - "Ти молодець, {name}!"
  - "Давай заробимо зірочки!"
  - "Я вірю в тебе!"
- **And** фраза показується в speech bubble + озвучка (для 🐣 обов'язково)
- **And** bounce анімація при тапі (scale 1→1.2→1, spring)
- **And** персонаж відповідає віковій групі (Коко/Бамбі/Ліса/Софі)
- **And** MVP: великий emoji (48sp) як placeholder, Phase 2: анімований SVG/Lottie

### AC-9: PIN — перший вхід (створення)
- **Given** PIN ще не встановлено (parentPin = null в settings)
- **When** натискаю "Батькам 🔒" вперше
- **Then** flow:
  1. Заголовок: "Створіть PIN-код"
  2. Ввести 4 цифри → dots заповнюються
  3. Заголовок змінюється: "Повторіть PIN-код"
  4. Ввести ті ж 4 цифри
  5. Якщо збігаються → PIN збережено (hashed) в MMKV, перехід до parent dashboard
  6. Якщо не збігаються → shake анімація dots, "PIN не збігається, спробуйте ще"
  7. Reset → вводити заново з кроку 1

### AC-10: PIN — подальші входи
- **Given** PIN встановлено
- **When** натискаю "Батькам 🔒"
- **Then** flow:
  1. Заголовок: "Введіть PIN-код"
  2. Ввести 4 цифри
  3. Якщо правильний → navigate до `/(parent)/dashboard`
  4. Якщо неправильний:
     - Shake анімація dots (translateX ±10, 3 рази, 300ms)
     - Dots скидаються
     - Counter невірних спроб +1
  5. Після 3 невірних спроб:
     - Lock на 5 хвилин (NFR-S3)
     - Показується: "Забагато спроб. Спробуйте через 5:00"
     - Таймер відліку (4:59, 4:58...)
     - Кнопки клавіатури disabled
     - При закритті/перезапуску — lock зберігається (MMKV timestamp)

### AC-11: Анімації входу на Hub
- **Given** перехід на Hub (з логіну або при запуску)
- **When** Hub завантажується
- **Then** анімації:
  1. Header: fadeIn + slideDown (translateY: -20→0, 300ms)
  2. Персонаж: fadeIn + scale (0.8→1, 400ms, delay 200ms)
  3. Speech bubble: fadeIn (delay 600ms, duration 300ms)
  4. Картки островів: staggered fadeIn + slideUp (кожна з delay +100ms)
  5. Bottom toolbar: fadeIn + slideUp (delay 800ms)
- **And** загальна тривалість входу: ~1.2 секунди
- **And** все одночасно якщо повертаємось з острова (без stagger, instant)

## Технічні нотатки
- Islands config: статичний масив, games per island — масив game IDs
- Responsive: `useWindowDimensions()` → calculate numColumns
- PIN зберігається hashed (SHA-256 або bcrypt) в MMKV, НЕ plaintext (NFR-S3)
- PIN lock timestamp: MMKV key `pin_lock_until`, перевіряється при відкритті modal
- Tab bar НЕ використовується — всі острови на одному Hub екрані
- Персонаж MVP: великий emoji (48sp) + Reanimated idle loop
- Level formula: `requiredXP(level) = 50 * level * (level + 1) / 2`
- Locked ігри: перевіряти по конфігу `island.games[i].implemented: boolean`
